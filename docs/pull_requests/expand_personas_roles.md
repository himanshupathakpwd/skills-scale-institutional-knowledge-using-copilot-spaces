# docs: Expand personas and roles with new supporting roles and interaction guidelines

## Summary
This pull request expands the project management documentation with comprehensive persona definitions and interaction guidelines to improve clarity and accountability across teams.

## Changes
- **Updated docs/octoacme-roles-and-personas.md**: Added detailed definitions for supporting roles (Business Analyst, QA Lead, DevOps Engineer, UX/UI Designer, Scrum Master, Stakeholder) with responsibilities, goals, and interaction patterns. Includes a RACI matrix showing role responsibilities across key project activities.
- **New docs/role-interaction-guidelines.md**: Created comprehensive guide on how different personas interact, communicate, and collaborate throughout the project lifecycle.

## Problem Addressed
The previous documentation only covered core roles (Developers, Product Managers, Project Managers) and lacked guidance on:
- Important supporting roles that are critical to project success
- Clear interaction patterns between personas
- Communication cadence and decision-making frameworks
- RACI accountability model

## Benefits
- **Improved Clarity**: Teams now understand all critical roles and their responsibilities
- **Reduced Friction**: Clear interaction patterns minimize miscommunication
- **Accountability**: RACI matrix clearly defines who is responsible, accountable, consulted, and informed
- **Scalability**: Documentation supports both simple and complex projects
